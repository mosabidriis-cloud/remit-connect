# REOS UI Rules

## Purpose

This document defines consistent interaction behavior across the application.

## General Principles

- Consistency first
- Minimize clicks
- Prevent user errors
- Show important information first
- Responsive by default

## Navigation

- Consistent sidebar
- Breadcrumbs on detail pages
- Current page clearly highlighted
- No dead-end navigation

## Buttons

- Primary action per screen
- Secondary actions grouped
- Destructive actions require confirmation
- Disabled buttons must explain why when appropriate

## Forms

- Required fields clearly marked
- Inline validation
- Preserve entered values on validation errors
- Clear success and error feedback

## Tables

- Search before scrolling
- Filters above tables
- Sortable columns where appropriate
- Sticky headers for long tables
- Empty state guidance

## Status Badges

- Pending
- Processing
- Completed
- Failed
- Cancelled

## Loading States

- Skeleton loaders preferred
- No layout shifting
- Visible progress for long operations

## Empty States

- Explain why no data exists
- Suggest the next action

## Confirmation Dialogs

Require confirmation for:

- Delete
- Cancel
- Complete irreversible actions

## Notifications

- Success
- Warning
- Error
- Information

## Accessibility

- Keyboard navigation
- Visible focus states
- Color is never the only indicator
- Sufficient contrast
