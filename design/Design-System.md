# REOS Design System

## Purpose

This document defines the visual language and UI standards for the entire REOS application.

## Design Principles

- Operations First
- Clarity Over Decoration
- Consistency Everywhere
- Safety Before Speed
- Accessibility by Default

## Visual Identity

- Product Name: REOS
- Full Name: Remit Exchange Operations System

## Typography

- Font: Inter
- Consistent heading hierarchy
- Readable body text

## Color Palette

- Primary: Blue
- Success: Emerald
- Warning: Amber
- Danger: Red
- Info: Sky
- Neutral: Slate

Reference Tailwind CSS utility colors only. Do not define custom hex values.

## Light Theme

- Gray background
- White cards
- Visible borders

## Dark Theme

- Slate background
- Slate cards
- Visible borders

## Border Radius

Use rounded-xl consistently.

## Shadows

Use shadow-sm by default. Avoid heavy shadows.

## Icons

Use Lucide icons only.

## Layout Principles

- Full-width layouts
- Summary cards at top
- Primary content before secondary content
- Consistent spacing
- Responsive by default

## Component Philosophy

All buttons, cards, tables, forms, badges, dialogs, and navigation components must follow this design system.
